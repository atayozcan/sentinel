# Sentinel-KDE 0.8.1

Maintenance release. No behavioural changes — dependency refresh plus
compatibility with the new `pam-bindings` 0.3 API. Upgrading from 0.8.0
is a drop-in: same PAM stack, same services, same dialog.

## Changes

- **`pam-bindings` 0.3.0** — `PamHandle::get_user` now takes `&mut self`;
  `check_agent_bypass` and the `resolve_user` helpers thread the mutable
  handle through accordingly. No effect on the auth path's behaviour.
- **Dependency updates** (via Dependabot): `log` 0.4.29 → 0.4.32, `zbus`
  and `zvariant` minor bumps, `clap_complete` minor bump.
- **CI / GitHub Actions** bumped to current major versions.

## Install

### Arch (AUR)

```bash
paru -S sentinel-kde       # stable
paru -S sentinel-kde-git   # main HEAD
```

### From source (any distro)

```bash
git clone https://github.com/atayozcan/sentinel-kde.git
cd sentinel-kde
sudo ./install.sh          # transactional; rolls back on error
```

### From the binary bundle

Download `sentinel-kde-0.8.1-<arch>-linux.tar.gz`, extract, then:

```bash
sudo SENTINEL_SKIP_BUILD=1 ./install.sh
```

No Rust toolchain needed.
