# Sentinel-KDE 0.8.0

First public release of the KDE Plasma sibling of [`sentinel`](https://github.com/atayozcan/sentinel) (which targets COSMIC). A Windows-UAC-style confirmation dialog for privilege escalation on **KDE Plasma (Wayland)** — when something asks for elevated rights (`pkexec`, a polkit action, `sudo`, `su`), Sentinel paints a Breeze/Kirigami dialog asking **Allow** or **Deny** instead of prompting for a password.

## What's in this release

- **`pam_sentinel.so`** — PAM module installed as `auth sufficient` on top of the distro's existing stack at `/etc/pam.d/{polkit-1,sudo,su}`. Denial or unreachability falls through to the normal password prompt — no lockout.
- **`sentinel-polkit-agent`** — systemd **user** service (`PartOf=graphical-session.target`) that registers as the session's polkit authentication agent and exposes a one-shot pre-approval on the **system** D-Bus (`org.sentinel.Agent`).
- **`sentinel-helper-kde`** — Kirigami/cxx-qt dialog. Renders as a fullscreen `zwlr-layer-shell-v1` overlay on Plasma + wlroots compositors, falling back to a normal xdg-toplevel on Mutter.

## Install

### Arch (AUR)

```bash
paru -S sentinel-kde       # stable
paru -S sentinel-kde-git   # main HEAD
```

### From source (any distro)

```bash
git clone https://github.com/atayozcan/sentinel-kde.git
cd sentinel-kde
sudo ./install.sh          # transactional; rolls back on error
```

### From the binary bundle

Download `sentinel-kde-0.8.0-<arch>-linux.tar.gz`, extract, then:

```bash
sudo SENTINEL_SKIP_BUILD=1 ./install.sh
```

No Rust toolchain needed.

## After install

```bash
systemctl --user mask  plasma-polkit-agent.service   # if you had it
systemctl --user start sentinel-polkit-agent.service
```

Pass `--no-sudo` to `install.sh` to leave `sudo`/`su` as plain password prompts and guard polkit only.

gh attestation verify sentinel-kde-0.8.0-x86_64-linux.tar.gz \

## Caveats

- Sentinel sits in the PAM auth path for polkit and (by default) `sudo`/`su`. A broken, denied, or disabled module **always falls through to the normal password** — no lockout. Still: keep `sudo -i` open during your first install until you've confirmed things work.
